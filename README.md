## About me

I work at the intersection of scientific research, data strategy and software
engineering.

My focus is turning complex scientific data into systems that are:

- discoverable and understandable;
- reproducible and traceable;
- validated through explicit metadata and quality rules;
- suitable for bioinformatics, analytics and AI/ML;
- maintainable as production-quality software.

My background spans computational biology, genomics, single-cell and spatial
omics, bioimaging, scientific data governance and cloud-based pipeline
engineering.

## Current areas of work

- Scientific data strategy and governance
- Metadata contracts, provenance and lineage
- Bioinformatics and scientific workflow engineering
- Data-quality and semantic validation
- FAIR and reproducible research objects
- AI-ready scientific datasets
- Data catalogues and scientific data products
- Open-source tools for biotechnology research

## How I approach engineering

I favour:

- explicit schemas and metadata contracts;
- typed interfaces and structured configuration;
- automated tests, linting and static analysis;
- deterministic and reproducible execution;
- modular command-line tools and APIs;
- documented architectural decisions;
- governance controls embedded directly into data workflows.

## Current direction

I am developing practical open-source tools that connect scientific software
engineering with data governance, reproducibility and responsible AI.

Particular interests include:

- benchmarking scientific data catalogues and governance controls;
- validating scientific datasets before downstream reuse;
- packaging analysis runs as traceable research objects;
- designing realistic synthetic environments for testing AI agents;
- improving interoperability across research data systems.
